Git

Workflow

Workflow-это процесс работы целой команды, он нужен для того, чтобы несколько людей из вашей команды параллельно работали над кодом,не мешая друг другу.Также с ним легко отслеживать все изменения коммитов,веток и т.д. и чтобы он не разрушал основной код(main), и оставался стабильным.

Rebase

Это чтобы переписать все данные с одной ветки в другую ветку.

Различие между git rebase и git merge если git merge нужен для слияния веток (с послендних коммитов),а git rebase чтобы переписать данные с одной ветки в другую

Fetch

Загружает изменения между удаленными репозиториями,не обьединяя их с друг с другом.

Различие между git fetch и git pull:git fetch не изменяет ветку,а Pull изменяет всё
